## 2. 重要な整理（設計の前提）

### 2.1 何を統合して、何を分離するか
- 意思決定: **単一SNN（Unified Core SNN）** に統合する
- 出力実行: **Actuation Drivers（薄いAPI呼び出し）** が担当する
- 安全停止: **Hard Stop は学習に依存させず、別経路で Driver を直接プリエンプトする**

### 2.2 ふだんの出力契約（Core→Driver）
- **Action Plan** = 任意テキスト + 低次元ベクトル $u$（要素を増やさない）
- $u$ の具体マッピング（音声の抑揚、操作の強さ等）は Driver 側に押し込める

### 2.3 割り込みの扱い
- **Hard Stop**: 例外なく止める（安全装置）
- **Soft Interrupt**: 止めろ命令ではなく、Core が「譲る/話し切る/待つ」等を選ぶための入力

## 5. よくある疑問

### Q. Reflex と Reservoir は単一SNNにまとめるべき？
- **まとめてもOK** です（概念的な領域分割を守れるなら）。
- ただし **Hard Stop は統合の外**です（学習や通常通信の混雑で遅れるのが最悪パターンのため）。

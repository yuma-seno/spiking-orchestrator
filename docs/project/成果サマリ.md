# 成果サマリ（初見向け）

この文書は、「このリポジトリで何をして、何ができるようになったか」を初見でも追えるようにまとめたものです。

最初に用語が辛い場合は、先に用語集を読むのがおすすめです:
- [docs/project/用語集.md](%E7%94%A8%E8%AA%9E%E9%9B%86.md)

## 1. 何を作っている？（一言）
SORCHは、音声対話で重要な **割り込み停止（Stop）** をまず“反射（Reflex）”で安全に実現し、
その上で“記憶（Hippocampal Reservoir）”を追加して会話テンポに適応していく構想です。

- Phase 1: まず **遅延（p95/p99）を実測** し、止められる土台を作る
- Phase 2: 次に **短期記憶（STP）** を設計し、文脈制御へ進む

## 2. どんな作業をした？（Phase別）
### Phase 1（遅延計測ベンチ / Reflex暫定）
- **遅延計測ベンチ** `sorch.bench.latency_bench` を実装
  - マイク入力 → 特徴量抽出 → 閾値判定 → stopイベント記録（JSONL）
  - 事故防止のため、デフォルトで生音を鳴らさない（無音出力モード）
- 実機（コンテナ内）で計測を行い、結果をサマリ化

成果物:
- 実測結果まとめ: [outputs/phase1/report/Phase1_測定結果.md](../../outputs/phase1/report/Phase1_%E6%B8%AC%E5%AE%9A%E7%B5%90%E6%9E%9C.md)
- 生ログ（JSONL）: [outputs/phase1/latency/](../../outputs/phase1/latency/)

### Phase 2（STPニューロン / MC探索）
- **STP付きニューロン**（Tsodyks–Markram）をSpikingJellyで実装し、pytestで検証
- **Memory Capacity（MC）** を計測し、パラメータ探索（coarse→fine）とレポート生成を整備
- 大規模探索（例: n=600 / repeats=5）を実行し、Top候補をレポート化

成果物:
- 大規模MCレポート例: [outputs/phase2/mc/reports/phase2_mc_refine_large_report.md](../../outputs/phase2/mc/reports/phase2_mc_refine_large_report.md)
- 実験CSV（生データ）: [outputs/phase2/mc/runs/](../../outputs/phase2/mc/runs/)

## 3. 何が「成果」なの？（どう嬉しい？）
- 仕様書のKPI（p95/p99等）に対して、**まず“測れる”状態**になった
- Phase 2では、STPパラメータ（tauF/tauDなど）を振って **MCの差が見える**状態で探索できる
- 生成物の保存先が固定され、初見でも
  - どのコマンドで
  - どのファイルが出て
  - どこを見れば良いか
 追えるようになった

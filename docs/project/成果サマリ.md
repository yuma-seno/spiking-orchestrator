## 2. いま何ができる？（短縮）

- Stop遅延の計測と集計（p95/p99の確認）を “ベンチ” として回せる
- STPリザーバと MC 探索の基盤があり、設定比較をレポート化できる
- RLS のオンライン更新をベンチで回せ、安定化の設計知見がある
- IPC（SPSC）と共有メモリ（Hard Stop / 低次元ベクトル）で、リアルタイム系の土台がある

---

## 3. 次に仕様として固めること（最短）

- Action Plan の最小スキーマと readout（出力チャネル→`text`/`u`）
- Driver の停止セマンティクス（無音化/停止/close/キャンセル）
- Hard Stop の E2E 計測（publish→無音化 / publish→物理停止）
